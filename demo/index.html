SQL Server blueprint

AshcolInventory

Static demo for the Inventory Database (SQL Server) course project: execution order, functional modules, and verification queries. Open this file in a browser, or serve the demo folder with any static server. This repo is separate from any other app database; other stacks may borrow ideas only. Deploy and test the real database in SSMS (SQL Server Management Studio) per your course requirements.

Build order (SSMS / Azure Data Studio)

  1. Run inventory/ddl/00_create_database.sql through 07_indexes.sql in order.
  2. Run all scripts in inventory/views/.
  3. Run all scripts in inventory/triggers/ (before seeds — movements populate stock).
  4. Run inventory/seeds/01_seed_reference_data.sql.
  5. Optional: inventory/transactions/ and inventory/warehouse/ for labs and OLAP.

Functional modules (10+)

Users & roles

Users, Roles, UserRoles — RBAC baseline.

Catalog

ProductCategories, Products — SKU master with soft delete.

Suppliers

Suppliers — vendor master linked to products.

Locations

Locations — warehouses, stores, branch vans.

Stock

StockLevels, StockMovements — ledger + on-hand (triggers enforce rules).

Customers

Customers — buyer master for sales.

Sales

SalesOrders, SalesOrderLines — normalized orders.

Purchasing

PurchaseOrders, PurchaseOrderLines — replenishment.

Invoicing

Invoices — billing linked to orders.

Reporting

Views (vw_ActiveProductCatalog, etc.) and optional dw star schema.

Verification (after seed)

Run in AshcolInventory and compare row counts.

SELECT COUNT(*) AS RoleCount FROM dbo.Roles;
SELECT COUNT(*) AS UserCount FROM dbo.Users;
SELECT COUNT(*) AS ProductCount FROM dbo.Products;
SELECT COUNT(*) AS StockLevelRows FROM dbo.StockLevels;

Views quick check

SELECT TOP 10 * FROM dbo.vw_ActiveProductCatalog;
SELECT TOP 10 * FROM dbo.vw_SalesOrderSummary;
